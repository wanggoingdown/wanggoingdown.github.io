Handler的用法

今天我来讲一下我们android常用的Handler，那么问题来了，啥是handler？字面意思是处理器。
那么我这个处理器主要是来处理什么的呢？

  1. 处理延时操作，例如X秒后执行某些特定操作（3秒后Finish当前Activity等）
  2. 经常需要子线程中需要进行耗时操作，例如访问网络数据，读取本地文件，然后在UI线程中刷新界面。

一般我们用Handler是这样用的

private Handler mHandler=new Handler(){
        @Override
        public void handleMessage(Message msg) {
            super.handleMessage(msg);
        }
    };

其实这样写乍一看没毛病，细心的同学就会发现这样会引发内存泄漏
由于mHandler是Handler的匿名内部类的实例，所以它持有外部类Activity的引用，我们知道消息队列是在一个Looper线程中不断轮询处理消息，而消息队列中的Message持有mHandler的实例引用，mHandler又持有Activity的引用，那么当这个Activity退出时，消息队列中还有未处理的消息或者正在处理的消息，将导致该Activity的内存资源无法及时回收，从而引发内存泄漏。
那么应该怎么写呢？

 private MyHandler mHandler = new MyHandler(MainActivity.this);

private static class MyHandler extends Handler {
    WeakReference<MainActivity> weakReference;

    public MyHandler(MainActivity activity) {
        weakReference = new WeakReference<MainActivity>(activity);
    }

    @Override
    public void handleMessage(Message msg) {
        super.handleMessage(msg);
        if (weakReference.get() == null) {
            Log.d(TAG, "该activity已经被销毁");
        }
    }
}

@Override
protected void onDestroy() {
    super.onDestroy();
    mHandler.removeCallbacksAndMessages(null);
}

还有一种写法是通过post方法发送Runnable对象，sendMessage发送Message
mHandler.postDelayed(new Runnable(){
    @Override
    public void run(){
    //TODO具体业务
    }
},1000);
new Thread(new Runnable(){
    @Override
    public void run(){
        handler.sendMessage(new Message(0));
   }
}).start();

以上两种就是常规Handler的用法
